# Atlas UI Component Library

Welcome to the **Atlas UI Component Library** documentation.

## What is Atlas?

Atlas is a standardized, opinionated front-end platform for building, scaling, and operating serious
web applications across teams. This is not a demo or a "we'll clean it up later" React app—it's
production-grade infrastructure for engineers.

## Component Library Philosophy

The Atlas UI library provides:

- **Accessibility baked in** - All components follow WAI-ARIA standards
- **Design tokens & variants** - Consistent styling using CVA (Class Variance Authority)
- **Dark mode support** - Every component works in both light and dark themes
- **Type-safe** - Full TypeScript support with comprehensive prop types
- **Tested** - All components have unit tests ensuring reliability

## Design System

Built on top of:

- **Radix UI** - Unstyled, accessible components
- **Tailwind CSS** - Utility-first CSS framework
- **Lucide React** - Beautiful, consistent icons

## Getting Started

Browse the components in the sidebar to see:

- **Interactive examples** - Try different props and states
- **Code snippets** - Copy-paste ready examples
- **Accessibility notes** - ARIA patterns and keyboard navigation
- **Dark mode** - Toggle between light and dark themes

## Usage in Your App

```tsx
import { Button, Card, Input } from "@atlas/ui";

function MyComponent() {
  return (
    <Card>
      <Input placeholder="Enter your name" />
      <Button>Submit</Button>
    </Card>
  );
}
```

## Principles

### Consistency

Every component follows the same patterns, making the codebase predictable and maintainable.

### Composition

Components are designed to work together seamlessly. Use them as building blocks for complex UIs.

### Quality

Automated gates prevent regressions. All components must pass linting, type checking, and testing.
